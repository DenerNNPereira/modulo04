métodos get e post para formulários. aula 03 capítulo 24 módulo 04

dentro de uma tag é atributo

por padrão o método de envio de dos formulários é get, se não especificado na tag form. para especificar um método usar a tag method método e configurar com o método escolhido, são dois métodos get e post. não tem diferença entre maiúsculo ou minúsculo quando adicionar post ou get no atributo method.

o método get envia os dados do formulário através da url. o método post envia os dados do formulário de forma interna, que pode ser acessado da seguinte forma: na ferramenta de inspeção entrar em network, depois em headers e no arquivo que foi enviado, após uma possível rolagem de tela os dados, depois de enviados clicado em no botão de submit, estarão visíveis. dependendo da versão do navegador depois de network clicar no arquivo enviado e em payload para encontrar os dados enviados do formulário. para proteger esses dados independente se get ou post é necessário usar o https que é um protocolo de transmissão de dados que possui criptografia.

quando usar cada método: get é pra quando os dados que o usuário digitou não são sensíveis não são senha ou cartão de crédito por exemplo, ou uma simples busca, neste caso uma busca fica compartilhável, o envio de dados através do método get é limitado a 3000 bites aproximadamente 3000 letras no campo da url. para o método post: usar o post quando os dados do formulário para dados sensíveis ou fotos ou arquivos ou mais de 3000 bites, pode ser apenas a questão de não deixar os dados visíveis na url.

regular expressions para definir um padrão de digitação usando o atributo pattern padrão inicia com ^ e termina com cifrão $. um RegEx para o pattern do input tipo tel pode ser escrito entre aspas duplas "" e se for dígitos pode ser \d que significa dígitos ou abre colchetes e adiciona o intervalo separado por traço - ficando: "^\d$" ou "^[0-9]$" em seguida, entre chaves {} colocar quantos dígitos serão necessários e para definir o formato adicionar um traço - ou o símbolo necessário e repetir, ficando: pattern="^[0-9]{4}-[0-9]{4}$" ou pattern="^\d{4}-\d{4}$". caso tenha a opção de uma quantidade ou outra de dígitos, por exemplo, uma vírgula entre valores que ficam dentro de chaves {} indica opções de quantidade de dígitos: pattern="^[0-9]{4,5}-[0-9]{4}$" ou pattern="^\d{4,5}-\d{4}$". para dizer que algum dado que vai ser digitado precisa estar entre parênteses em uma RegEx a combinação de símbolos \(valoraqui)\ indica abrir e fechar parênteses: pattern="^\([0-9]{2}\)[0-9]{4}-[0-9]{4}$" ou pattern="^\(d{2}\)\d{4}-\d{4}$"

a configuração de display configurada como none: display: none; no css faz com que o o seletor css que receber esta configuração seja exibido na tela do navegador normalmente porém não será impresso. para configurar o que será impresso, essa configuração deve estar no arquivo css direcionado às configurações de impressão.

um seletor css configurado com content e um conteúdo de texto entre aspas simples '' : content: 'texto'; não será exibido na tela do navegador, porém será impresso. para configurar o que será impresso, essa configuração deve estar no arquivo css direcionado às configurações de impressão.

nos links de css os atributos de mídia do tipo screen e print são os mídia type.
