inconvenientes do iframe. aula 06 capítulo 22 módulo 04

frame não é muito utilizado porém confinua em funcionamento. iframe é bastante utilizado, mas pode ter algum incoveniente. mecanismos de buscas podem ter dificuldade em identificar conteúdos dentro de iframes, afetando a eficiência da busca. softwares leitores de tela podem ter dificuldade para ler o conteúdo dentro de iframes afetando a acessibilidade de pessoas com alguma difilculdade de visão que usam esses softwares. a navegação dentro de iframes pode ser um pouco complicada, voltar página, target etc, ou mesmo a responsividade do site que estiver dentro do site. iframes podem ser muito inseguros na questão de trazer conteúdo completo para um iframe, um site que inseguro ou que eventualmente esteja capturando dados ou facilitando ataques pode trazer essa insegurança para o site onde o iframe está.


tornando iframes mais seguros. aula 07 capítulo 22 módulo 04

uma forma de deixar o iframe mais seguro é usar o parâmetro sandbox dentro da tag iframe. por padrão sandbox é sandbox que bloqueia o envio de tudo dentro do iframe. sandbox ligado sandbox é o máximo desse parâmetro para segurança. a opção allow-same-origin no sandbox permite o uso de formulários dentro do iframe, desde que seja da mesma origem do iframe. no atributo sandbox pode ser adicionado vários valores de configurações, desde que tenha um espaço entre elas e fique entre parênteses. para permitir formulários dentro do iframe, no parâmetro sandbox usar: allow-forms permitir formulários. para permitir o conteúdo como srcdoc usar: allow-scripts permitir scripts também libera ou bloqueia a execução de scripts em javascript

outra maneira de dar mais segurança é configurar o parâmetro referrerpolicy, que tem como valor padrão no-referrer-when-downgrade, com o valor: no-referrer, faz com que o conteúdo dentro do iframe não irá coletar nenhum tipo de dado do usuário. no-referrer em conjunto de sandbox é o máximo de segurança para o usuário

criando a página inicial do projeto. aula 03 capítulo 23 módulo 04

quando criar o link para o css e o arquivo não estiver criado, segurar a tecla control ctrl e clicar em cima do nome do arquivo na linha do link para o css, uma mensagem de erro vai aparecer e dar a opção de criar esse arquivo, marcar criar e o arquivo será criado e linkado. pegar o tamanho da imagem pelo gimp, 311 por 627 telefone.

zerar a margem e o padding de forma geral com margin: 0px; e padding: 0px; no seletor * asterístico. nos seletores body e html colocar a altura height e largura width para 100% da viewport com height: 100vh; e width: 100vw; vh para altura e vw para largura.

o seletor para o body receberá o background fixo sem repetição alinhado ao topo e no centro cobrindo todo o espaço da viewport ficando: background: url('../imagens/fundo-madeira.jpg') no-repeat top center; background-size: cover; background-attachment: fixed;

o seletor main ocupa 100% da altura da viewport e possui posicionamento relativo: position: relative; height: 100vh;

a section com id telefone recebe a imagem de um celular com tela preta usando background e com o endereço da imagem dentro do atributo url. precisa ter posicionamento absoluto para que seja colocado em qualquer ponto da tela. o alinhado a 50% pelo topo e esquerda. o parâmetro transform com o atributo translate, configurado para -50% e -50% para que o ponto do alinhamento seja o centro da imagem ou da caixa onde a imagem está. sem isso o alinhamento será pelo canto superior esquerdo da imagem ou caixa onde a imagem está.

criando os botões para redes sociais. aula 05 capítulo 23 módulo 04

nos seletores css quando não se adiciona o sinal de maior > para relação pai e filho, indica que o último em qualquer hierarquia dentro do inicial, exemplo: se a hierarquia é section link img, no seletores css pode adicionar section img sem sinal de maior > aqui: uma imagem img em qualquer posição de hierarquia porém dentro de um section, se adicionado o sinal de maior e linha de hierarquia deve ser completa: section > link > img aqui: uma imagem img que esteja dentro de um link, que esteja dentro de um section.

para deixar imagem quadradas redondas usar border-radius com a configuração de 50% ao invés de um valor em pixels.

uma forma de fazer os links ficarem empilhados, um em cima do outro, é usar a tag br após cada link. para alinhar a direita, uma forma bem simples é configurar um seletor css para o box onde estão e configurar o atributo text-aling para a direção desejada, neste caso direita rigth.

a configuração no css box-sizing configurada como border-box faz com que a borda que for adicionada ao elemento, imagem ou caixa, seja considerada como parte o elemento, caso um hover adicione borda a um elemento, por exemplo, essa borda já está sendo considerada no elemento, só não estará visível até que o hover seja ativado.

para fazer o elemento ter um movimento dentro do hover usar a configuração no parâmetro transform com translate com dois valores, sendo o primeiro para direita e esquerda e o segundo acima e abaixo em relação ao eixo x e y então valores negativos direciona para esquerda e acima e valores positivos direciona para direita e abaixo. caso o elemento que receba essas configurações tenha sombra o ideal é que esta acompanhe o movimento. para a sombra acompanhar o movimento basta que um box-shadow seja adicionado ao hover e no box-shadow dentro do hover seja adicionado do valor sem hover mais o valor do translate que do mesmo hover.
